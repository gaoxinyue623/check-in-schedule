# 安卓安装包打包指南

您的项目已经配置好 Capacitor 安卓环境，现在需要生成最终的 APK 安装包。由于您的系统中未检测到 Java 环境，无法自动完成最后一步构建，请按照以下步骤操作。

## 前置准备

请确保您已安装 **Android Studio**（推荐）或配置好 Java JDK 环境。

## 方法一：使用 Android Studio（推荐）

这是最简单的方法，适合大多数用户。

1. **打开 Android Studio**。
2. 选择 **Open**（打开项目），然后找到您项目目录下的 `android` 文件夹选中打开：
   `前端AI项目/check-in-schedule/android`
3. 等待 Android Studio 自动同步项目（首次打开可能需要下载一些 SDK 工具，耗时较长）。
4. 同步完成后，点击顶部菜单栏的 **Build** -> **Build Bundle(s) / APK(s)** -> **Build APK(s)**。
5. 等待构建完成。
6. 构建成功后，右下角会出现提示，点击 **locate** 即可找到生成的 APK 文件。
   - 文件通常位于：`android/app/build/outputs/apk/debug/app-debug.apk`

您可以将这个 `app-debug.apk` 发送到安卓手机上进行安装测试。

## 方法二：使用命令行（需要配置 Java）

如果您已经安装了 Java JDK 17+ 和 Android SDK，可以使用命令行构建：

1. 打开终端，进入 android 目录：
   ```bash
   cd android
   ```
2. 运行构建命令：
   ```bash
   ./gradlew assembleDebug
   ```
3. 构建完成后，APK 文件位于：
   `app/build/outputs/apk/debug/app-debug.apk`

## 注意事项

- 当前生成的是 **Debug** 版本，可以安装在大多数手机上用于测试。
- 如果需要发布到应用商店，需要生成 Signed APK（签名包），请在 Android Studio 中选择 **Build** -> **Generate Signed Bundle / APK** 并按照指引操作。

## 常见问题

**Q: 只有 index.html 显示空白？**
A: 请确保 `dist/index.html` 中的资源路径是正确的（Capacitor 环境下通常能正常处理）。如果遇到白屏，请连接 Android Studio 查看 Logcat 日志。

**Q: 更改代码后如何更新？**
A: 每次修改 Vue 代码后，请运行以下命令同步到安卓项目：
   ```bash
   npm run build
   npx cap sync
   ```
   然后重新构建 APK。
