## 📄 核心文件说明

### `src/main.js`
应用的入口文件，负责：
- 创建Vue应用实例
- 注册Pinia状态管理
- 注册Vue Router路由
- 挂载应用到DOM

### `src/App.vue`
根组件，包含：
- RouterView用于显示路由页面
- 引入全局样式

### `src/assets/main.css`
全局样式文件，定义了：
- CSS变量（颜色、间距等）
- 通用样式（按钮、卡片、标题等）
- 动画效果
- 响应式设计

### `src/stores/schedule.js`
Pinia状态管理，管理：
- **状态数据**：
  - `weekSchedule`: 一周的计划数据
  - `currentDayIndex`: 当前选中的日期
  - `totalStars`: 总星星数
  - `gifts`: 礼品列表
  
- **核心方法**：
  - `initWeekSchedule()`: 初始化一周的计划
  - `toggleTask()`: 切换任务完成状态
  - `updateTask()`: 更新任务信息
  - `addTask()`: 添加新任务
  - `deleteTask()`: 删除任务
  - `redeemGift()`: 兑换礼品
  - `saveToLocalStorage()`: 保存数据到本地
  - `loadFromLocalStorage()`: 从本地加载数据

### `src/views/HomeView.vue`
主页面组件，包含：
- **头部区域**：
  - 标题
  - 星星计数器
  
- **周导航**：
  - 7天的日期选择器
  - 显示每天获得的星星数
  
- **进度卡片**：
  - 今日完成进度条
  - 今日获得的星星数
  
- **任务列表**：
  - 任务分类筛选
  - 任务列表展示
  - 任务完成打勾
  - 任务编辑和删除
  
- **礼品兑换**：
  - 礼品展示
  - 兑换功能
  
- **模态框**：
  - 添加任务表单
  - 编辑任务表单

### `src/router/index.js`
路由配置，定义：
- 路由模式（HTML5 History）
- 路由路径和对应的组件

## 🔄 数据流

```
用户操作 → 触发事件 → 调用Store方法 → 更新状态 → 保存到LocalStorage → 视图更新
```

### 示例：完成任务
1. 用户点击任务复选框
2. 触发 `@change` 事件
3. 调用 `toggleTask(dayIndex, taskId)`
4. 更新任务的 `completed` 状态
5. 更新 `totalStars`
6. 调用 `saveToLocalStorage()`
7. Vue响应式系统自动更新视图

## 🚀 性能优化

1. **代码分割**：使用Vite的自动代码分割
2. **懒加载**：路由组件按需加载
3. **本地存储**：减少不必要的计算
4. **响应式优化**：使用computed计算属性

## 🔧 可扩展性

### 添加新功能
1. **新页面**：在 `src/views/` 添加新组件，在路由中注册
2. **新组件**：在 `src/components/` 添加可复用组件
3. **新状态**：在 `src/stores/` 添加新的store
4. **新样式**：在 `src/assets/` 添加样式文件

### 自定义主题
修改 `src/assets/main.css` 中的CSS变量即可改变整体配色。

### 添加新任务类别
在 `HomeView.vue` 的 `categories` 数组中添加新类别。
